# FitForge MVP仕様

## コンセプト

健康、ダイエット、筋トレ、ランニング、HYROX、マラソンを一つにまとめ、体重目標に対して「今日なにをすればいいか」まで出すiOSアプリ。

## 初期ターゲット

- 20代から40代の日本人
- ダイエットだけでなく筋トレやランニングも続けたい人
- 食事管理アプリ、筋トレアプリ、ランニングアプリを分けて使うのが面倒な人
- HYROXやマラソンに挑戦したいが、体重管理も同時にしたい人

## MVPで必須の機能

1. 今日のカロリー収支を見る
2. 食事を文章で入力して、AI風にカロリーとPFCを推定する
3. 筋トレの種目、重量、回数、セット数を記録する
4. 重量を上げるタイミングを提案する
5. ランニング、HYROX、マラソンの距離、時間、消費カロリーを記録する
6. 体重目標を設定する
7. 週次、月次、年次の理論体重変化と実績を比較する
8. HealthKitから体重、歩数、消費カロリー、ワークアウトを読むための権限導線を用意する
9. 30秒チェックインで「食事ざっくり」「運動ざっくり」「体調」「気分」を残せる
10. ローカル保存で、アプリを閉じても記録が消えない
11. 初回オンボーディングで目的、起床時刻、体重目標、運動頻度、食事記録の細かさを設定する
12. 食事AIは推定後にユーザーがカロリー/PFC/食事名を修正してから保存する
13. HealthKitの歩数、活動カロリー、基礎代謝、最新体重を生活日ベースで手動同期できる
14. SwiftDataへ保存したデータを起動時にAppStoreへ反映する
15. 食事AI API URLを設定でき、未設定時はローカル推定にフォールバックする
16. HealthKitの過去7日/30日サマリーを同期できる
17. 目的と食事記録スタイルに応じて、ホームと食事画面を出し分ける

## 日本人向けのUX方針

- 日本語を標準言語にしつつ、文言キーを通して多言語化できる設計にする
- 筋トレ種目は日本語名を正として、英語名と別名も持つカタログを用意する
- 1日の区切りは0時固定ではなく、ユーザーの起床時刻を基準にする
- 「理論上は何kg減るはず」と「実際は何kg減った」を並べて表示する
- 断定しすぎず、「目安」「推定」「次回候補」という表現を使う
- 「未達」「警告」よりも「今日はここまででOK」「軽めに続けましょう」を使う
- 入力は1画面で完結させる
- カロリーだけでなくタンパク質を見やすくする
- 競技者向けすぎず、一般ユーザーにも分かる言葉にする

## 注意表現

- AIの食事分析は推定値であり、医療・栄養指導そのものではない
- 急激な減量を推奨しない
- 体調不良、痛み、摂食不安がある場合は専門家へ相談する
- HealthKitデータはユーザー許可の範囲でのみ読み取る

## サブエージェントから反映したMVP判断

- 健康・ダイエット: 週0.25〜0.5kg程度の現実的な減量ペースを基本にする
- トレーナー: 筋トレは「記録 → 伸びの可視化 → 次回候補」を核にする
- HYROX/マラソン: 単なる距離記録ではなく、ペース、RPE、種別、弱点メモを残す
- 一般ユーザー: 30秒チェックインを用意し、詳細入力しない日も継続扱いにする
- iOS開発者: MVPでもローカル保存を入れ、HealthKitと手入力の出所を分ける
- マーケター: 日本の24時間ジム利用者に向けて「迷わず今日やることが分かる」を価値にする
- 管理栄養士: AI推定は断定せず、ユーザー修正前提にする
- プライバシー担当: 身体データ、食事、写真はセンシティブ情報として扱う

## 生活日設計

日付の区切りは暦上の0時ではなく、ユーザーが設定した起床時刻を基準にする。初期値は日本人ユーザー向けに5:00。たとえば5:00設定の場合、深夜1:00の食事や運動は前日の生活ログとして扱う。

この設計により、夜勤、早朝勤務、深夜トレーニング、飲み会後の食事などを自然に記録できる。
